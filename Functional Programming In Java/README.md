## Chapter 1: Introduction to Functional Programming
- **Why Use Functional Programming?**
  - Promotes immutability, statelessness, and easier reasoning about code.
  - Reduces side effects and improves testability.
- **Declarative Programming**
  - Focuses on what to do, not how to do it.
  - Example: Using `Stream` API for data processing.
- **Immutability**
  - Data cannot be changed after creation.
  - Example: Using `final` variables and immutable collections.
- **Examples:**
  - Pure functions, stateless computations, and simple lambda expressions.
